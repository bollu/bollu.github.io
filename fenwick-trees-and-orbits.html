§ Fenwick trees and orbits

I learnt of a nice, formal way to prove the correctness of Fenwick trees in terms of orbits that I wish to reproduce here. One can use a Fenwick tree to perform cumulative sums Sum(n)≡∑i=0nA[i]Sum(n) \equiv \sum_{i=0}^n A[i]Sum(n)≡∑i=0n​A[i], and updates Upd(i,v)≡A[i]+=vUpd(i, v) \equiv A[i] += vUpd(i,v)≡A[i]+=v. Naively, cumulative sums can take O(n)O(n)O(n) time and updates take O(1)O(1)O(1) time. A Fenwick tree can perform both in log⁡(n)\log(n)log(n). In general, we can perform any monoid-based catenation and update in log⁡(n)\log(n)log(n), as long as our queries start from the 0th index till some index nnn. So we can only handle monoidal fold queries of the form ∑i=0n\sum_{i=0}^n∑i=0n​ and point updates.

§ correctness

We wish to analyze the operations Query(q)≡∑i=1qa[i]Query(q) \equiv \sum_{i=1}^q a[i]Query(q)≡∑i=1q​a[i], and Update(i,val)≡a[i] += valUpdate(i, val) \equiv a[i]~\texttt{+=}~valUpdate(i,val)≡a[i] += val. To do this, we are allowed to maintain an auxiliary array ddd which we will manipuate. We will stipulate the conditions of operations on ddd such that they will reflect the values of QueryQueryQuery and UpdateUpdateUpdate, albeit much faster. We will analyze the algorithm in terms of orbits. We have two operators, one for update called UUU, and one for query called QQQ. Given an index iii, repeatedly applying the query operator gives us the indeces we need to read and accumulate from the underlying array aaa to get the total sum a[0..i]a[0..i]a[0..i]:
  • Query(i)=∑id[Qi(q)]Query(i) = \sum_i d[Q^i(q)]Query(i)=∑i​d[Qi(q)]
Given an index uuu, repeatedly applying the update operator UUU gives us all the indeces we need to add the change to update:
  • Update(i,val)=∀j ,d[Uj(i)] += valUpdate(i, val) = \forall j~, d[U^j(i)]~\texttt{+=}~ valUpdate(i,val)=∀j ,d[Uj(i)] += val
For query and update to work, we need the condition that:
  • q≥u  ⟺  ∣{Qi(q) : i∈N}∩{Ui(u) : i∈N}∣=1q \geq u \iff \left\vert \{ Q^i(q)~:~ i \in \mathbb{N} \} \cap \{ U^i(u)~:~i \in \mathbb{N} \} \right\vert = 1q≥u⟺∣∣∣​{Qi(q) : i∈N}∩{Ui(u) : i∈N}∣∣∣​=1
That is, if and only if the query index qqq includes the update location uuu, will the orbits intersect. The intuition is that we want updates at an index uuu to only affect queries that occur at indeces q≥uq \geq uq≥u. Hence, we axiomatise that for an update to be legal, it must the orbits of queries that are at indeces greater than it. We will show that our operators:

# calculate orbits of query and update in fenwick tree

def lsb(i): return i & (-i)
def U(i): return i + lsb(i)
def Q(i): return i - lsb(i)
def orbit(f, i):
    s = set()
    while i not in s and i > 0 and i < 64:
        s.add(i); i = f(i)
    return s

§ Case 1: q=uq = uq=u

We note that QQQ always decreases the value of qqq, and uuu always increases it. Hence, if q=uq = uq=u, they meet at this point, and ∀i,j≥1,Qi(q)≠Uj(u)\forall i, j \geq 1, \quad Q^i (q) \neq U^j(u)∀i,j≥1,Qi(q)=Uj(u). Hence, they meet exactly once as required.
