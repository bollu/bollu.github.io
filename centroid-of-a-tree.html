§ Centroid of a tree

  • Do not confuse with Center of a tree , which is a node vvv that minimizes the distance to all other nodes: maxw∈Vd(v,w)max_{w \in V} d(v, w)maxw∈V​d(v,w). This can be found by taking the node that is the middle of a diameter.
  • The centroid of a tree is a node such that no child has over floor(n/2) of the vertices in the tree.

§ Algorithm to find centroid of a tree

  • Root tree arbitrarily at rrr
  • Compute subtree sizes with respect to this root rrr.
  • Start from root. If all children of root rrr have size less than or equal to floor(n/2), we are done. Root is centroid.
  • If not, some child ccc [for child, contradiction ] has size strictly greater than floor(n/2).
  • The total tree has nnn vertices. ccc as a subtree has greater than floor(n/2)vertices. Thus the rest of the tree (ie, the part under rrr that excludes ccc) has strictly less than floor(n/2) vertices.
  • Let us in our imagination reroot the tree at this child ccc. The childen of ccc continue to have the same subtree size. The old node rrr, as a subtree of the new root ccc, has size strictly ness than floor(n/2) vertices.
  • Now we recurse, and proceed to analyze c.
  • This analysis shows us that once we descend from r -> c, we do not need to analyze the edge c -> r if we make c the new candidate centroid.

  • That is, it is the vertex that on removal induces subtrees, such that the size of the largest component is smallest amongst all nodes.
