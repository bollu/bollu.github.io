§ Crash course on DCPO: formalizing lambda calculus

In lambda calculus, we often see functions of the form λx→x(x)\lambda x \rightarrow x(x)λx→x(x). We would like a way to associate a "natural" mathematical object to such a function. The most obvious choice for lambda calculus is to try to create a set VVV of values which contains its own function space: (V→V)⊆V(V \rightarrow V) \subseteq V(V→V)⊆V. This seems to ask for a set whose cardinality is such that ∣V∣∣V∣=∣V∣|V|^|V| = |V|∣V∣∣V∣=∣V∣, which is only possible if ∣V∣=1|V| = 1∣V∣=1, ie VVV is the trivial set {∗}\{ * \}{∗}. However, we know that lambda calculus has at least two types of functions: functions that terminate and those that don't terminate. Hence the trivial set is not a valid solution. § Difference between DCPO theory and Domain theory

  • A DCPO (directed-complete partial order) is an algebraic structure that canbe satisfied by some partial orders. This definition ports 'continuity'to partial orders.
  • A domain is an algebraic structure of even greater generality than a DCPO.This attempts to capture the fundamental notion of 'finitely approximable'.
  • The presentation of a domain is quite messy. The nicest axiomatization ofdomains that I know of is in terms of information systems.One can find an introduction to these in the excellent book 'Introduction to Order Theory' by Davey and Priestly

§ CPOs

  • Given a partial order (P,≤)(P, \leq)(P,≤). assume we have a subset Q⊆PQ \subseteq PQ⊆P. A least upper bound uuu of QQQ is an element that is the smallest element in PPPwhich is larger than every element in QQQ.
  • A subset QQQ of PPP is called as a chain if its elements can be put into order.That is, there is a labelling of elements of QQQ into q1,q2,…,qnq1, q2, \dots, qnq1,q2,…,qn suchthat q1≤q2≤⋯≤qnq1 \leq q2 \leq \dots \leq qnq1≤q2≤⋯≤qn.

§ CCPOs

  • A partially ordered set is called as a chain complete partial order ifeach chain has a least upper bound.
  • This is different from a lattice, where each subset has a least upper bound.
  • Every ccpo has a minimal element given by completion(∅)=⊥completion(\emptyset) = \botcompletion(∅)=⊥.

  • Composition of monotone functions is monotone.
  • The image of a chain wrt a monotone function is a chain.

§ Continuous function

  • A function is continous if it is monotone and preserves all LUBs. This isonly sensible as a definition on ccpos, because the equation defining it is:lub . f = f . lub, where lub: chain(P) \rightarrow P. However, for lubto always exist, we need P to be a CCPO. So, the definition of continuousonly works for CCPOs.
  • The composition of continuous functions of chain-complete partiallyordered sets is continuous.

§ ≤\leq≤ as implication

We can think of b≤ab \leq ab≤a as b  ⟹  ab \implies ab⟹a. That is, bbb has more information than aaa, and hence implies aaa.
